[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at the system architecture. (pause)
First, every query enters our workflow and is immediately filtered by the security checkpoint node, which scrubs PII and checks for prompt injections.
If the query is safe, the workflow routes to the coordinator agent, which acts as the orchestrator.
The coordinator agent delegates tasks to specialized sub-agents: the medication agent and the symptom agent, or uses MCP tools directly.
If a medication change is requested, the system registers the details in the workflow state and transitions to the human approval checkpoint node.
Here, a RequestInput pause occurs, blocking any automated database changes until a human coordinator verifies the request.
Once approved, the post-approval handler executes the changes by running the medication agent to call the Stdio-based FastMCP server. (pause)
